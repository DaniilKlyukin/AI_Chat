# AI Engineering Assistant

Это автономный ИИ-агент с веб-интерфейсом, предназначенный для помощи в программировании, системном администрировании и управлении файлами. Агент работает локально, используя Ollama, и имеет доступ к файловой системе и терминалу вашего компьютера (внутри выделенной папки).

## 🚀 Основные возможности

- **Автономный цикл работы:** Агент обдумывает задачу (`<thought>`), выбирает инструмент, выполняет действие и анализирует результат.
- **Инструменты (Tools):**
    - Чтение и запись файлов (целиком).
    - Выполнение консольных команд (терминал).
    - Просмотр дерева проекта и поиск файлов.
    - Навигация по директориям.
- **Интерфейс:** Современный SPA на Tailwind CSS с подсветкой кода и настройками модели в реальном времени.
- **Память:** История переписки и состояние текущей папки сохраняются в базе данных SQLite.

---

## 🛠 Предварительные требования

1.  **Python 3.10+**
2.  **Ollama** (скачайте с [ollama.com](https://ollama.com))
3.  **Модель:** По умолчанию используется `qwen3:8b` (или любая другая, например `llama3`, `mistral`).

---

## 📦 Установка и настройка

### 1. Подготовка Ollama
Убедитесь, что Ollama запущена, и скачайте необходимую модель:
```bash
ollama pull qwen3:8b
```

### 2. Клонирование и установка зависимостей
Перейдите в папку `backend` и создайте виртуальное окружение:
```bash
cd backend
python -m venv .venv

# Для Windows:
.venv\Scripts\activate
# Для Linux/macOS:
source .venv/bin/activate

pip install -r requirements.txt
```
*(Если файла requirements.txt нет, установите основные библиотеки: `fastapi uvicorn litellm sqlalchemy aiosqlite pydantic-settings`)*

### 3. Конфигурация (.env)
Создайте файл `.env` в корневом каталоге проекта (рядом с папками `backend` и `frontend`):

```env
# Какую модель использовать (формат litellm: ollama/название)
MODEL=ollama/qwen3:8b

# Адрес, где запущена Ollama
API_BASE=http://localhost:11434
```

### 4. Структура папок
Проект автоматически создаст папку `backend/storage`. Именно внутри этой папки агент будет выполнять все действия по умолчанию. Это сделано для безопасности.

## 🚦 Запуск приложения

Из папки `backend` запустите сервер:

```bash
python app/main.py
```
Или через uvicorn:
```bash
uvicorn app.main:app --reload
```

После запуска откройте браузер по адресу:
👉 **http://127.0.0.1:8000**

## 🖥 Как пользоваться

### Интерфейс
- **Sidebar (слева):** Отображает текущую рабочую директорию и ID сессии.
- **Настройки агента:**
    - **Temperature:** (0.1 — 1.0) — точность ответов. Для кода лучше ставить 0.1.
    - **Max Iterations:** Сколько шагов (инструментов) агент может запустить за один ваш запрос.
- **Чат (центр):** Пишите команды на естественном языке (на русском или английском).

### Примеры команд
- *"Создай структуру папок для проекта на FastAPI."*
- *"Проверь содержимое файла agent.py и объясни, как работает функция run_cycle."*
- *"Выполни команду `pip list` и скажи, установлена ли библиотека sqlalchemy."*
- *"Найди во всех файлах текст 'litellm' и выведи список этих файлов."*

## 📂 Структура проекта

- **`backend/app/core/agent.py`**: "Мозг" агента. Реализует цикл ReAct (Thought -> Action -> Observation). Обрабатывает "галлюцинации" модели, если она пишет JSON текстом.
- **`backend/app/services/tools.py`**: Реализация функций (чтение, запись, терминал).
- **`backend/app/main.py`**: API сервер на FastAPI, раздача статики фронтенда.
- **`backend/app/db/models.py`**: Схема БД SQLite для хранения сообщений.
- **`frontend/`**: Чистый JavaScript и HTML. Для Markdown используется `marked.js`, для кода — `highlight.js`.

## ⚠️ Безопасность

1.  **Выполнение команд:** Агент имеет доступ к вашему терминалу через `subprocess`. Не просите его выполнять команды, в которых вы не уверены.
2.  **Перезапись:** Агент всегда старается прочитать файл перед записью, чтобы не удалить лишнее, но всегда делайте бэкапы важных данных.
3.  **Изоляция:** По умолчанию агент работает в `backend/storage`, но технически может выйти за ее пределы, если дать прямую команду через абсолютный путь.

---

## 🛠 Возможные проблемы

- **Ошибка 500 (Connection Error):** Проверьте, запущена ли Ollama и правильно ли указан `API_BASE` в `.env`.
- **ИИ не вызывает инструменты:** Убедитесь, что используемая модель поддерживает **Function Calling**. Модели семейства Qwen и Llama3 отлично справляются с этим.
- **Бесконечный цикл:** Если агент зациклился, уменьшите `Max Iterations` в боковой панели. В коде реализована защита от повторного вызова одного и того же инструмента с теми же аргументами.

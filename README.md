# Primat AI — Виртуальный помощник кафедры ПМиИТ

**Primat AI** — это полнофункциональное веб-приложение на базе искусственного интеллекта, разработанное для помощи студентам и сотрудникам кафедры ПМиИТ. Система позволяет не только общаться с ИИ-ассистентом, но и автоматически генерировать файловые структуры, программный код и обрабатывать загруженные документы (PDF, DOCX, TXT).

## 🚀 Основные возможности

-   **🤖 Интеллектуальный чат:** Использование современных локальных моделей (по умолчанию `deepseek-r1:8b`) через Ollama.
-   **📁 Работа с файлами (Инструменты):**
    -   **Автогенерация:** Агент может самостоятельно создавать папки и файлы на основе вашего запроса.
    -   **Экспорт:** После создания проекта ИИ формирует ZIP-архив, который можно скачать прямо из чата.
-   **📄 Чтение документов:** Поддержка загрузки и анализа файлов форматов `.pdf`, `.docx`, а также текстовых файлов в кодировках UTF-8 и CP1251.
-   **🛠 Гибкое управление:**
    -   Переключатель использования инструментов (Tools) в реальном времени.
    -   История переписки с привязкой к сессии (сохраняется в памяти сервера).
    -   Автоматическая очистка временных файлов и истории.
-   **🎨 Современный интерфейс:** Адаптивный дизайн в стиле мессенджера, поддержка Markdown, подсветка синтаксиса кода и Drag-and-Drop для файлов.

---

## 🛠 Предварительные требования

1.  **Python 3.10+**
2.  **Ollama** (скачайте с [ollama.com](https://ollama.com))
3.  **Модель:** По умолчанию используется `deepseek-r1:8b`. Вы можете использовать любую другую модель, поддерживающую Function Calling.

---

## 📦 Установка и настройка

### 1. Подготовка Ollama
Запустите Ollama и скачайте модель:
```bash
ollama pull deepseek-r1:8b
```

### 2. Клонирование и установка зависимостей
Перейдите в папку проекта и создайте виртуальное окружение:

```bash
# Создание окружения
python -m venv .venv

# Активация (Windows)
.venv\Scripts\activate
# Активация (Linux/macOS)
source .venv/bin/activate

# Установка библиотек
pip install fastapi uvicorn litellm pydantic-settings python-multipart pypdf python-docx jinja2
```

### 3. Конфигурация
Создайте файл `.env` в корневом каталоге (рядом с папкой `backend` и `frontend`):

```env
MODEL=ollama/deepseek-r1:8b
API_BASE=http://localhost:11434
TEMPERATURE=0.7
```

*Вы также можете отредактировать файл `system_prompt.txt` в корне проекта, чтобы изменить личность или инструкции ассистента.*

---

## 🚦 Запуск приложения

Запустите сервер из корневой директории проекта:

```bash
python backend/app/main.py
```

После запуска приложение будет доступно по адресам:
-   Локально: [http://127.0.0.1:8000](http://127.0.0.1:8000)
-   В локальной сети: `http://[ваш-ip]:8000` (IP отобразится в консоли при старте).

## 🖥 Как пользоваться

### Отправка сообщений и файлов
-   Введите запрос в поле ввода. Чтобы отправить файл, нажмите на иконку скрепки или просто **перетащите файл (Drag-and-Drop)** в окно браузера.
-   Текст из PDF или Word документов будет автоматически извлечен и передан ИИ вместе с вашим вопросом.

### Генерация кода и проектов
Попросите ИИ: *"Создай проект на Python для анализа данных с папкой 'data' и файлом 'main.py'"*.
1.  ИИ вызовет функции `create_directory` и `create_file`.
2.  В чате появится уведомление о выполненных действиях.
3.  Справа или в конце сообщения появится кнопка **"Скачать ZIP"** с вашим готовым проектом.

### Управление инструментами
В верхней панели есть переключатель **Tools**. Если он выключен, ИИ будет работать в режиме обычного чат-бота без возможности создавать файлы на вашем диске.

## 📂 Структура проекта

```text
├── backend/
│   ├── app/
│   │   ├── api/
│   │   │   └── endpoints.py      # Маршруты API (чат, история, очистка)

│   │   │   ├── ai_service.py     # Логика взаимодействия с Ollama и парсинг функций
│   │   │   ├── file_service.py   # Обработка документов и создание ZIP
│   │   │   └── history_service.py # Управление историей сообщений
│   │   └── main.py               # Точка входа, запуск сервера и статика
│   └── storage/                  # Временное хранилище сгенерированных файлов
├── frontend/
│   ├── index.html                # Интерфейс приложения
│   ├── script.js                 # Логика чата, загрузки файлов и UI-событий
│   └── style.css                 # Стилизация и анимации
├── system_prompt.txt             # Системная роль ИИ
└── .env                          # Переменные окружения
```

## ⚠️ Безопасность и ограничения

1.  **Изоляция:** Все файлы создаются в строго определенной директории `backend/storage/{session_id}`. Ассистент не имеет доступа к системным файлам за пределами этой папки.
2.  **Очистка данных:** При нажатии кнопки "Очистить" на сервере удаляются как история сообщений, так и все физически созданные файлы данной сессии.
3.  **Локальный запуск:** Все данные обрабатываются локально на вашем компьютере. Информация не передается сторонним компаниям (при использовании локальной Ollama).
